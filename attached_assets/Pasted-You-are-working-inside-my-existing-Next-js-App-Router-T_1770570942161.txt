You are working inside my existing Next.js (App Router, TypeScript) project.

GOAL
Enhance my existing Service JSON-LD by adding:
1) A short, factual, one-sentence description for each Service
2) A serviceType field
3) A category field that matches my Google Business Profile categories

This should improve AI and search understanding while keeping schema clean and non-duplicative.

CONTEXT (DO NOT CHANGE)
- I already have a site-wide LocalBusiness schema with:
  - @id: https://ocmoldpros.com/#business
  - aggregateRating + review ONLY on LocalBusiness
  - Services nested under hasOfferCatalog → Offer → Service
- JsonLd injection is handled by a reusable server component
- Schema generation lives in app/lib/schema.ts
- Services are currently generated from a SERVICES array

DO NOT:
- Create new top-level schema entities
- Duplicate LocalBusiness or Organization
- Duplicate reviews or ratings
- Change UI, layout, navigation, or routes

ONLY modify schema data structures.
