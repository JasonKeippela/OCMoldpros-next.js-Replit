CRITICAL SEO PROBLEMS TO FIX (do all of these)
════════════════════════════════════════

PROBLEM 1 — openGraph does not deep-merge in Next.js
The root layout.tsx defines a global openGraph block. Page-level metadata exports
do NOT override individual openGraph fields — they must re-declare the ENTIRE
openGraph object or they will inherit the root layout's og:url (the homepage).
Every page is currently showing og:url = https://ocmoldpros.com regardless of
which page it is. This is causing Google to treat all pages as duplicates of
the homepage and is the primary reason 80 of 100 pages are not indexed.

FIX: Every page.tsx must export its own complete openGraph object with the
correct url field for that specific page. Never rely on partial inheritance.

PROBLEM 2 — Shared, generic metadata across all pages
Every page currently shares identical og:title, og:description, og:url,
twitter:title, twitter:description, and meta keywords from the root layout.
Google sees this as a duplicate content signal.

FIX: Every page.tsx must have a fully unique metadata export including:
- title (unique per page, keyword-rich, under 60 chars)
- description (unique per page, action-oriented, 140–160 chars)
- keywords (unique per page, 5–8 locally relevant terms)
- alternates.canonical (correct full URL for this page)
- openGraph.title (can match title)
- openGraph.description (can match description)
- openGraph.url (MUST be the full URL of THIS specific page)
- openGraph.type, openGraph.locale, openGraph.siteName, openGraph.images
- twitter.card, twitter.title, twitter.description, twitter.images

PROBLEM 3 — Missing schema markup
No pages currently have structured data. For a local health-adjacent services
business, this significantly reduces E-E-A-T signals.

FIX: Add the following schema to the pages listed:
- LocalBusiness schema: every page (import from a shared helper)
- Service schema: all service and category pages
- FAQPage schema: all category pages and location pages
- BreadcrumbList schema: all pages except homepage
